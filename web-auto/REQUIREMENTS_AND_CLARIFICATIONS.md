# web-auto 需求与澄清文档（v2.3）

更新时间：2026-03-10
位置：`J:\project_code\auto-label\web-auto`
状态：已按当前实现刷新

## 1. 范围
- 只允许修改 `web-auto` 和 `sam3-api`
- 不修改 `sam3` 官方仓库代码
- 所有推理统一走远程 `sam3-api`
- `web-auto` 只负责项目管理、提示组织、结果预览与标注写回

## 2. 图片标注已确认能力
- 一键推理：当前图片 + 已选类别
- 一键推理：全部图片 + 已选类别
- 点提示：严格一次只生成一个实例
- 框提示：只支持正负框，不支持点框混合
- 预览结果默认不全选，支持手动勾选、删除、全选后提交
- 保留“清空当前标注”
- 支持删除当前图片，并同时删除对应标注文件（若存在）
- 项目允许中途新增图片
- 项目管理页支持拖拽或选择图片上传到当前项目 `image_dir`
- 全图文本推理支持可调 `batch_size`
- 顶栏支持“智能过滤”，用于扫描并合并近重复同类目标
- 批量文本推理与全图范例传播支持任务进度条

## 3. 视觉能力拆分
本次重点调整：把“框选修正”和“范例语义”彻底拆开，避免同一个入口既做纯视觉又做文本+范例，造成行为不明确。

### 3.1 纯视觉预览
入口：`预测预览`

规则：
- 点选时走 `points` 纯视觉提示
- 框选时走 `boxes` 纯视觉提示
- 不携带 `active_class`
- 不结合文本 prompt
- 只生成预览，不直接写入正式标注
- 自动分割支持两种可选模式：
  - `纯视觉预览`
  - `当前图范例分割`
  - 其中 `当前图范例分割` 仅对框选有效

用途：
- 局部修正
- 单目标交互式分割
- 不希望文本类别影响结果时使用

### 3.2 当前图范例分割
入口：`范例分割`

规则：
- 必须选择 `active_class`
- 必须至少提供 1 个正样本框
- 负样本框可选
- 调用远程 `sam3-api /v1/semantic/infer`
- 只作用于当前图片
- 只生成预览，不直接写回

用途：
- 当前图 concept segmentation
- 明确希望“类别文本 + 框范例”共同作用时使用

### 3.3 全图集范例传播
入口：`范例传播: 全图集`

规则：
- 必须选择 `active_class`
- 必须至少提供 1 个正样本框
- 负样本框可选
- 调用远程 `sam3-api /v1/semantic/infer_batch`
- 长期固定为“全图集”范围，不提供子集选择
- 直接写回，不做“先批量预览再提交”
- 仅替换目标类别，不覆盖其他类别

用途：
- 把当前图片的视觉范例传播到整个图片项目

## 4. 当前接口约定

### 4.1 web-auto 对外接口
- `POST /api/infer`
  - 文本推理、点提示、纯视觉框提示
- `POST /api/infer/preview`
  - 文本/点/纯视觉框的预览版
- `POST /api/infer/example_preview`
  - 当前图范例分割预览
- `POST /api/infer/batch`
  - 全图文本批量推理
  - `web-auto` 会按 `batch_size` 分块转发到远端 `/v1/infer_batch`
- `POST /api/infer/batch_example`
  - 全图集范例传播
- `POST /api/infer/jobs/start_batch`
  - 启动全图文本批量推理任务
- `POST /api/infer/jobs/start_batch_example`
  - 启动全图范例传播任务
- `GET /api/infer/jobs/{job_id}`
  - 查询当前任务进度

### 4.2 sam3-api 远程接口
- `POST /v1/infer`
  - 文本推理
  - 点提示
  - 纯视觉框提示
- `POST /v1/infer_batch`
  - 图片文本多图 batch
- `POST /v1/semantic/infer`
  - 当前图范例语义分割（文本 + 正负框）
- `POST /v1/semantic/infer_batch`
  - 全图集范例传播（文本 + 源图范例框）

### 4.3 当前多图 batch 范围
- 当前只在“全部图片 + 已选类别”的文本推理上启用官方多图 batch
- 前端提供 `batch_size` 输入，用于控制每次送往远端的图片数
- `batch_size` 过大可能导致远端显存不足
- 点提示、纯视觉框预览、当前图范例分割，当前都不走这条多图 batch
- 全图集范例传播在 `web-auto` 侧按 `batch_size` 分块调用远端语义 batch，以便显示进度

## 4.4 输入分辨率规则
- 文本/点/纯视觉框链路默认输入分辨率为 `1008`
- 当前图范例分割 / 全图集范例传播链路默认使用 Ultralytics 侧默认 `imgsz`
- `web-auto` 不再暴露手动分辨率设置，避免提供无效配置项

## 5. 类别规则
- 兜底类别使用 `unknown`
- 允许保留 API 返回的原始类别，不强制映射到项目类别
- `selectedClasses` 只影响文本一键推理
- `active_class` 用于：
  - 点提示命名
  - 当前图范例分割
  - 全图集范例传播
- `active_class` 不再影响“框选纯视觉预览”

## 6. 预览与提交规则
- `预测预览` / `范例分割` 都只进入预览区
- 预览区支持：
  - 单条删除
  - 全选预览
  - 清空预览
  - 完成标注
- `完成标注` 只提交当前勾选的预览项
- 默认不全选，避免误写入

## 6.1 智能过滤规则
- 入口：顶栏 `智能过滤`
- 先在弹窗内选择规则，再分析项目内所有图片并展示“可合并图片 + 候选目标数”
- 支持两种过滤方式：
  - 同类合并
  - 主从类别合并（目标类别 + 来源类别）
- 覆盖率阈值可调
- 面积计算方式可选：
  - 实例面积
  - 目标框面积
- 当前覆盖判定中的相交部分仍按 `bbox` 计算；面积计算方式只影响大/小目标排序和覆盖率分母
- 确认后会直接写回标注文件

## 7. 项目管理规则
- 项目管理页保留单一入口：`新增数据`
- 点击后弹出上传框
- 支持拖拽图片上传
- 支持点击按钮选择多个图片文件
- 文件保存到当前项目 `image_dir`
- 同名文件自动改名为 `_uploadN`
- 上传完成后自动重扫并追加进项目
- 不删除旧图片记录，不影响既有标注

## 8. 交互规则
- 左键按当前工具操作
- `Alt + 左键` 可随时拖拽浏览
- 不依赖右键平移，避免浏览器手势冲突
- 缩放和位移按图片隔离保存，不在不同图片之间串联
- 按钮支持悬停提示，用于说明用途与副作用
- 顶栏状态区下方单独显示批量任务进度条
- 进度条当前覆盖：
  - 全图文本批量推理
  - 全图集范例传播

## 9. 当前实现结论
- “预测预览”已经改成纯视觉，不再偷偷带文本
- “范例分割”和“范例传播”作为独立入口存在
- 自动分割新增下拉模式：
  - `纯视觉预览`
  - `当前图范例分割`
  - 范例模式仅在框选时触发
- 所有语义/传播推理仍统一走远程 `sam3-api`
- 全图文本推理已改为远端多图 batch，`web-auto` 负责分块提交并逐图写回结果

## 10. 当前无阻塞澄清项
如果后续还要继续细化，只剩两个可能的产品方向：
1. 是否增加“仅框范例、无文本”的官方风格当前图 concept segmentation
2. 是否增加“仅视觉传播、无文本类别”的全图集传播模式

当前版本未实现这两项，现有范例能力仍然是“文本 + 正负框”。
