# web-auto Frontend Requirements

本文档描述当前 `web-auto` 后端已经支持、前端需要承接的现有功能需求。目标不是重新设计业务，而是把现有能力完整落到一个新的独立前端里。

## 1. Product Scope

前端需要覆盖两类项目：

- 图片标注项目
- 视频标注项目

前端不需要实现后端业务逻辑，所有数据读写、推理、导出、任务控制都通过 `web-auto` API 完成。

## 2. Global Requirements

- 采用前后端分离架构
- 支持通过配置填写 `web-auto` API 地址
- 支持通过配置填写 `sam3-api` 地址
- 所有长任务必须显示真实进度，而不是假进度
- 所有错误必须展示后端原始错误信息
- 对 1 万张以上图片的数据集保持可用
- 图片列表必须采用分页或虚拟滚动，不能一次加载全部
- 切图时只请求单图标注与单图文件
- 前端应支持任务恢复，页面刷新或重新进入项目后能继续轮询活动任务

## 3. Project List Page

需要的能力：

- 查看项目列表
- 创建图片项目
- 创建视频项目
- 删除项目
- 打开项目
- 展示项目基础统计

创建项目表单：

- `name`
- `project_type`
- 图片项目时填写 `image_dir`
- 视频项目时填写 `video_path`
- `save_dir`
- `classes_text`

## 4. Image Project Workspace

### 4.1 Layout

建议至少包含以下区域：

- 顶部工具栏
- 左侧图片列表
- 中央图片画布
- 右侧类别、标注、推理、过滤、导出区域

### 4.2 Project Metadata

需要展示：

- 项目名
- 项目 ID
- 图片目录
- 已标注数
- 未标注数
- 总图片数

### 4.3 Image List

需要的能力：

- 分页或虚拟滚动加载图片
- 上一张 / 下一张
- 根据图片状态显示已标注 / 未标注
- 删除当前图片
- 刷新图片目录
- 导入图片
- 上传图片

### 4.4 Image Viewer

需要的能力：

- 显示原图
- 缩放
- 平移
- 适应窗口
- 显示已有标注
- 显示预览标注

### 4.5 Annotation Editing

至少需要支持：

- 查看已有标注列表
- 删除单个标注
- 保存当前图片标注
- 清空当前图片标注
- 追加新标注

建议保留的数据结构：

- `id`
- `class_name`
- `raw_label`
- `score`
- `bbox`
- `polygon`

## 5. Prompt And Inference

### 5.1 Single Image Text Inference

需要支持：

- 选择一个或多个类别
- 当前图文本推理
- 单图推理完成后直接显示结果

### 5.2 Point / Box Preview

需要支持：

- 正点
- 负点
- 正框
- 负框
- 仅预览，不立刻保存
- 用户可决定是否提交预览结果

### 5.3 Example Preview

需要支持：

- 当前图范例分割预览
- 输入来源框
- 可选 `pure_visual`
- 结果只做预览，允许用户再提交

### 5.4 Batch Text Inference

需要支持：

- 全图文本批量推理
- 启动任务
- 显示真实进度
- 停止
- 继续
- 继续时允许调整 `threshold`、`batch_size`、`sam3-api`

### 5.5 Example Batch Propagation

需要支持：

- 以当前图为源图做范例传播
- 启动任务
- 显示真实进度
- 停止
- 继续
- 继续时允许调整参数

## 6. Intelligent Filter

这是当前图像工作区的重要工具，新前端需要完整支持。

### 6.1 Filter Config

配置项：

- `merge_mode`
- `coverage_threshold`
- `area_mode`
- `canonical_class`
- `source_classes`

### 6.2 Preview

需要支持：

- 分析预览
- 真实进度条
- 显示当前处理进度
- 显示预览结果摘要
- 显示受影响图片列表

### 6.3 Apply

需要支持：

- 只有完成预览后才能确认合并
- 确认合并必须复用同一次预览的 `preview_token`
- 如果预览已过期，要提示用户重新预览
- 合并执行时显示真实进度

### 6.4 Visualization

当前后端已支持预览与确认的任务化，但更强的图上可视化还没有做。新前端建议补：

- 将删除的目标高亮
- 将改类的目标高亮
- 在受影响图片上快速跳转

## 7. Export

图片项目需要支持：

- COCO
- JSON
- YOLO

并支持：

- 导出 bbox
- 导出 mask
- 指定导出目录

视频项目需要支持：

- JSON 导出

## 8. Video Project Workspace

视频项目不要复用图片项目页面，要独立工作区。

需要的能力：

- 播放视频
- 获取视频文件流
- 如有需要手动触发 H.264 转码
- 选择 `prompt_mode`
- 文本提示或框提示
- 设置 `threshold`
- 设置 `imgsz`
- 设置 `segment_size_frames`
- 设置起止帧
- 启动视频传播任务
- 停止
- 继续
- 实时进度显示
- 查看当前帧结果

## 9. Task UX Requirements

所有长任务都需要统一交互：

- 任务启动后立即显示状态
- 轮询任务详情接口
- 页面刷新后能恢复任务显示
- 出错时显示后端错误
- 完成后自动刷新当前项目/当前图

涉及的任务类型：

- 文本批量推理
- 范例传播
- 智能过滤预览
- 智能过滤确认合并
- 视频传播

## 10. Performance Requirements

针对大规模数据集，前端必须遵守：

- 不一次拉全量图片
- 不一次拉全量标注
- 图片列表虚拟滚动
- 标注按需加载
- 文件按需请求
- 本地缓存必须有上限

## 11. Suggested Build Order

建议实现顺序：

1. 项目列表页
2. 图片项目工作区基础版
3. 单图标注与单图推理
4. 批量推理任务
5. 智能过滤
6. 导出
7. 视频项目工作区

## 12. Current Non-Goals

当前后端没有强制要求前端实现这些高级功能：

- 多人协作
- 登录鉴权
- 权限系统
- 审核流
- 版本回滚

如果后续需要，可以在独立前端上继续扩展。
