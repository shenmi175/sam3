# web-auto Frontend Requirements

本文档用于指导你重写 `web-auto` 前端。目标不是给出抽象功能列表，而是给出接近页面原型的交互要求，做到“照着文档就能开始搭页面和状态管理”。

后端约束：

- 现有后端为纯 API 服务
- 具体接口见 `DEVELOPER_API_GUIDE.md`
- 前端不负责业务计算，只负责状态管理、画面展示、用户交互和 API 编排

## 1. 总体设计原则

- 图片项目和视频项目必须使用两套工作区，不要共用同一页面
- 所有长任务都必须任务化展示，不能只转菊花
- 所有主要操作都要有状态反馈，不允许用户点击后“无事发生”
- 大规模图片数据集必须默认按分页和虚拟列表设计，不能先做全量加载再优化
- 布局以桌面端优先，同时保证 1366 宽度下可用；移动端只要求可查看，不要求完整生产效率

## 2. 页面结构总览

至少包含 3 个顶层页面：

1. 项目列表页
2. 图片标注页
3. 视频标注页

建议另有全局层：

- 全局提示条或 Toast
- 全局确认框
- 全局错误弹窗

## 3. 项目列表页

### 3.1 布局

页面采用上下布局：

- 顶部为页面标题区
- 中部左侧为“创建项目”表单
- 中部右侧为“项目列表”
- 页面底部可选放简短状态提示

建议宽度分配：

- 左侧创建区：30%-35%
- 右侧项目列表区：65%-70%

### 3.2 顶部标题区

从左到右：

- 页面标题：`web-auto`
- 副标题：`SAM3 标注工作台`
- 右侧可预留“后端健康状态指示”

### 3.3 创建项目区

表单字段从上到下排列：

1. `项目名称`
2. `项目类型`
3. `图片目录` 或 `视频路径`
4. `输出目录`
5. `初始类别`
6. `创建项目` 按钮

字段细节：

- `项目类型` 为二选一：`图片项目` / `视频项目`
- 选择 `图片项目` 时显示 `图片目录`
- 选择 `视频项目` 时显示 `视频路径`
- `初始类别` 使用多行文本框，支持逗号、换行混输

按钮关系：

- `创建项目` 位于该表单最底部，单独一行
- `创建项目` 点击后，按钮进入 loading 状态，直到返回结果

### 3.4 项目列表区

每个项目卡片至少显示：

- 项目名称
- 项目 ID
- 项目类型
- 图片数或帧数
- 已标注数
- 未标注数
- 图片目录或视频路径

每个项目卡片右侧按钮从左到右：

- `打开`
- `删除`

按钮关系：

- `打开` 为主按钮
- `删除` 为危险按钮
- 删除前必须二次确认

## 4. 图片标注页总体布局

图片标注页采用“三栏布局 + 顶部工具条 + 顶部进度区”。

从上到下：

1. 顶部导航条
2. 顶部操作条
3. 任务进度区
4. 主内容区

主内容区从左到右：

1. 左侧图片信息与图片列表栏
2. 中央画布工作区
3. 右侧类别与标注操作栏

### 4.1 顶部导航条

从左到右：

- `返回项目页`
- 当前项目名称
- 当前项目类型标签：`图片项目`
- 右侧预留 `浅色/深色切换`

### 4.2 顶部操作条

该区域是一整行，按逻辑分组，不要散乱堆叠。

从左到右建议顺序：

1. `web-auto API 地址`
2. `sam3-api 地址`
3. `测试 API`
4. `阈值`
5. `批大小`
6. `当前图推理`
7. `全图文本批推`
8. `范例分割`
9. `范例传播`
10. `智能过滤`
11. `导出`

控件关系要求：

- `测试 API` 紧跟在 `sam3-api 地址` 后面
- `阈值` 与 `批大小` 放在中部，属于全局推理参数
- `当前图推理`、`全图文本批推`、`范例分割`、`范例传播` 为一组
- `智能过滤`、`导出` 为另一组

### 4.3 顶部任务进度区

位于操作条正下方，独占一整行。

这个区域不是只给某一个任务用，而是统一承载当前图片页上的长任务状态。

从上到下：

1. 第一行：任务标题 + 任务状态文本 + 右侧操作按钮
2. 第二行：进度条
3. 第三行：任务参数摘要

第一行左侧内容示例：

- `当前任务：全图文本批推`
- `处理中 41/200`
- `41/200 (20.5%)`

第一行右侧按钮：

- `停止`
- `继续`

按钮关系：

- 任务运行中显示 `停止`
- 任务暂停后显示 `继续`
- 二者可以同时占位，但不可同时可点击

第三行参数摘要至少显示：

- 模式
- 范围
- 类别
- 阈值
- batch
- 范例传播时补充 `当前类别`、`纯视觉`、`源图`

### 4.4 左侧栏

左侧栏从上到下：

1. 当前项目信息卡
2. 图片翻页按钮行
3. 当前图片信息
4. 图片列表

#### 当前项目信息卡

显示：

- 项目名
- 项目 ID
- 图片目录
- 总图片数
- 已标注数
- 未标注数

#### 图片翻页按钮行

从左到右：

- `上一张`
- `下一张`
- `删除当前图`

#### 当前图片信息

显示：

- 当前序号，如 `41 / 12000`
- 当前文件名
- 当前图片状态：`已标注` / `未标注`

#### 图片列表

每行必须显示：

- 小圆点或色条表示标注状态
- 文件名
- 当前行高亮

交互要求：

- 只加载当前分页
- 滚动时触发下一页加载
- 点击行切图
- 切图前若存在未保存的手工修改，必须有明确策略
  
建议策略：
  
- 自动保存，或
- 弹出确认框：`保存并切换` / `放弃修改`

### 4.5 中央画布区

中央区从上到下：

1. 画布工具条
2. 显示开关与缩放信息条
3. 画布本体

#### 画布工具条

从左到右：

- `点选`
- `框选`
- `拖动`
- `正样本`
- `负样本`
- `撤销提示`
- `清空提示`
- `预览分割`
- `提交预览`
- `清空预览`

按钮关系：

- `点选/框选/拖动` 为互斥工具
- `正样本/负样本` 为互斥标注极性
- `预览分割` 只对当前提示生效
- `提交预览` 只有存在预览结果时可点
- `清空预览` 会清掉本次预览，不影响已保存标注

#### 显示开关与缩放信息条

从左到右：

- `显示掩码`
- `显示框`
- `点/框后自动分割`
- 自动分割模式下拉框
- `缩放百分比`
- `缩小`
- `适应窗口`
- `放大`

自动分割模式下拉：

- `视觉预览`
- `当前图范例分割`

### 4.6 右侧栏

右侧栏从上到下：

1. 类别管理区
2. 当前类别区
3. 标注列表区
4. 智能过滤入口区
5. 导出入口区

#### 类别管理区

从上到下：

- `类别输入框`
- `添加类别` 按钮
- 类别列表

类别列表每一项从左到右：

- 复选框：是否参与文本推理
- 类别名
- 序号或计数标签
- 删除按钮

按钮关系：

- 添加类别按钮紧跟输入框
- 类别删除按钮位于每一行最右侧

#### 当前类别区

用于点/框预览与范例相关功能。

从上到下：

- `当前类别` 标题
- 当前类别单选或高亮选择列表
- `纯视觉范例` 开关

说明文案必须显式写出：

- 开启 `纯视觉范例` 时，范例提示不带文本类别，仅结果落到当前类别

#### 标注列表区

每条标注卡片至少显示：

- 类别名
- 置信度
- bbox 摘要
- 是否带 polygon

每条标注卡片右侧按钮：

- `定位`
- `删除`

底部按钮：

- `保存当前图标注`
- `清空当前图标注`

## 5. 图片标注页主流程

### 5.1 打开项目

步骤：

1. 进入图片标注页
2. 读取项目元数据
3. 加载第一页图片
4. 自动打开第一张或上次停留图片
5. 同时检查是否存在未完成批量任务
6. 若存在，恢复顶部任务进度区

### 5.2 单图文本推理

步骤：

1. 用户勾选一个或多个类别
2. 设置阈值
3. 点击 `当前图推理`
4. 后端返回结果
5. 结果直接写入当前图标注
6. 右侧标注列表刷新
7. 画布立即显示

### 5.3 点/框预览

步骤：

1. 用户选择 `点选` 或 `框选`
2. 用户选择 `正样本` 或 `负样本`
3. 在画布上输入提示
4. 点击 `预览分割`，或在开启自动分割时自动触发
5. 结果进入“预览层”，不能直接混入正式标注
6. 用户点击 `提交预览` 后才写入正式标注

### 5.4 当前图范例分割

步骤：

1. 用户选择当前类别
2. 用户画正框作为范例提示
3. 可选打开 `纯视觉范例`
4. 点击 `范例分割`
5. 当前图返回预览结果
6. 用户决定是否提交

### 5.5 全图文本批推

步骤：

1. 用户勾选类别
2. 设置阈值和批大小
3. 点击 `全图文本批推`
4. 顶部任务进度区显示真实进度
5. 用户可点击 `停止`
6. 停止后允许修改参数
7. 点击 `继续`
8. 任务结束后刷新当前图和项目统计

### 5.6 范例传播

步骤：

1. 用户在当前图上给出范例框
2. 选择当前类别
3. 可选打开 `纯视觉范例`
4. 点击 `范例传播`
5. 顶部进度区显示任务进度
6. 允许停止与继续
7. 完成后刷新当前图和项目统计

## 6. 智能过滤弹窗

智能过滤不要做成页面跳转，必须做成弹窗。

### 6.1 弹窗布局

从上到下：

1. 标题栏
2. 配置区
3. 规则说明区
4. 进度区
5. 预览列表区
6. 底部操作按钮区

### 6.2 配置区

第一行从左到右：

- `过滤方式`
- `覆盖率阈值`
- `面积计算`

第二行：

- 当 `过滤方式 = 主从类别合并` 时，显示 `目标类别`

第三行：

- 当 `过滤方式 = 主从类别合并` 时，显示 `来源类别` 多选列表

### 6.3 规则说明区

必须是实时文案，不是静态说明。

要求：

- 根据当前配置实时改写说明
- 预览完成后必须追加一句：`确认合并会直接复用这次预览结果，不再重复扫描`
- 如果预览已过期，必须明确提示用户重新预览

### 6.4 进度区

从上到下：

1. 状态文本 + 右侧数值进度
2. 进度条

示例：

- `分析 381/12000: xxx.jpg`
- `381/12000 (3.2%)`

### 6.5 预览列表区

每行显示：

- 候选删除数量
- 图片路径
- 覆盖对数量
- 改类数量

列表项点击后建议支持：

- 跳转到对应图片

### 6.6 底部按钮区

从左到右：

- `分析预览`
- `确认合并`
- `取消`

按钮关系：

- 未完成预览前，`确认合并` 禁用
- 预览无结果时，`确认合并` 禁用
- 执行任务期间，配置与按钮整体禁用

### 6.7 智能过滤主流程

步骤：

1. 点击 `智能过滤`
2. 弹出弹窗
3. 用户配置规则
4. 点击 `分析预览`
5. 弹窗内显示真实进度
6. 完成后展示摘要和受影响图片
7. 用户点击 `确认合并`
8. 后端复用同一次预览的 `preview_token`
9. 弹窗继续显示写回进度
10. 完成后关闭弹窗，刷新当前图与项目统计

## 7. 导出弹窗

导出同样使用弹窗。

布局从上到下：

1. `导出格式`
2. `保存内容`
3. `导出目录`
4. 提示文案
5. 底部按钮

字段要求：

- `导出格式`：`COCO` / `JSON` / `YOLO`
- `保存内容`：`BBox` / `Mask`
- `导出目录`：文本输入

底部按钮从左到右：

- `确认导出`
- `取消`

## 8. 视频标注页总体布局

视频页不要照搬图片页，建议采用“两栏布局 + 顶部播放器控制区 + 顶部任务进度区”。

从上到下：

1. 顶部导航条
2. 顶部参数控制条
3. 顶部视频任务进度区
4. 主内容区

主内容区从左到右：

1. 左侧视频播放与帧浏览区
2. 右侧视频任务与标注区

### 8.1 顶部参数控制条

从左到右：

- `sam3-api 地址`
- `测试 API`
- `阈值`
- `imgsz`
- `segment_size_frames`
- `prompt_mode`
- `起始帧`
- `结束帧`
- `提示帧`
- `启动传播`

### 8.2 左侧视频区

从上到下：

1. 视频播放器
2. 播放控制条
3. 当前帧信息
4. 当前帧预览图或覆盖层

播放控制条从左到右：

- `播放/暂停`
- `上一帧`
- `下一帧`
- 当前时间 / 当前帧号
- `转码 H.264`

### 8.3 右侧视频区

从上到下：

1. 类别选择
2. 提示方式区域
3. 当前任务信息
4. 当前帧标注列表
5. 视频标注读写区

提示方式区域：

- 当 `prompt_mode = text` 时显示类别多选
- 当 `prompt_mode = boxes` 时显示框提示输入说明和当前类别

### 8.4 视频任务进度区

结构与图片页任务进度区一致，但字段补充：

- 当前帧号
- 已处理帧数
- fps
- 平均每帧耗时

右侧按钮：

- `停止`
- `继续`

## 9. 视频页主流程

### 9.1 打开视频项目

步骤：

1. 获取项目元数据
2. 获取视频任务状态
3. 加载视频流
4. 拉取当前视频标注
5. 若存在运行中任务，恢复进度区

### 9.2 启动视频传播

步骤：

1. 选择提示模式
2. 设置阈值、imgsz、segment_size_frames
3. 设置起止帧和提示帧
4. 点击 `启动传播`
5. 顶部进度区实时更新
6. 当前帧标注区同步展示结果

### 9.3 暂停与继续

步骤：

1. 点击 `停止`
2. 任务状态变为 `paused`
3. 用户可修改阈值、imgsz 等参数
4. 点击 `继续`
5. 后端从剩余帧继续

### 9.4 视频标注读写

步骤：

1. 前端通过 `/video/annotations` 拉整段数据
2. 用户修改某些帧标注
3. 调用 `/video/annotations/save`
4. 可选择 `replace_all=true` 或 `replace_all=false`

## 10. 全局状态与提示

前端必须有统一状态反馈系统。

### 10.1 顶部或底部状态栏

建议固定在页面底部或右上角区域。

至少显示三类消息：

- 成功
- 普通提示
- 错误

必须覆盖的提示场景：

- API 测试成功/失败
- 项目创建成功/失败
- 图片/视频加载成功/失败
- 推理成功/失败
- 任务停止/继续
- 智能过滤预览过期
- 导出成功/失败

### 10.2 Loading 状态

以下按钮点击后必须有 loading 或禁用态：

- 创建项目
- 当前图推理
- 全图文本批推
- 范例分割
- 范例传播
- 智能过滤分析预览
- 智能过滤确认合并
- 导出
- 启动视频传播
- 视频继续

## 11. 性能要求

### 图片项目

- 图片列表必须分页和虚拟化
- 标注只按当前图加载
- 不允许打开项目时一次性加载全量图片详情
- 本地图片缓存必须设置上限
- 标注缓存必须设置上限

### 视频项目

- 以流式播放为主，不要先下载全视频
- 帧图像按需拉取
- 大型标注 JSON 读取后应按帧索引组织，避免每次线性扫描

## 12. 推荐实现顺序

1. 项目列表页
2. 图片标注页基础框架
3. 单图标注与单图推理
4. 图片批量任务进度区
5. 智能过滤弹窗
6. 导出弹窗
7. 视频标注页
8. 视频标注读写和视频传播任务

## 13. 当前非目标

当前不要求前端实现：

- 登录鉴权
- 多人协作
- 审核流
- 角色权限
- 操作历史回滚

如果后续需要，可以在新前端中扩展。
